## Cross-Phase Integration Gaps

### P0 — Breaks Primary User Journey

**1. Holdout generation never triggered**
- **Affected:** HOLD-01, HOLD-02, HOLD-03, HOLD-05, LLM-06, WEB-05
- `generateHoldoutScenarios` (engine/holdout/generator.ts) is exported and tested but never called from any tRPC procedure, CLI command, or Inngest handler
- After `approveSummary` crystallizes a seed, nothing triggers holdout generation
- Vault is always empty → `sealHoldouts` always throws "No approved holdout entries"
- **Impact:** Full pipeline (crystallize → seal → decompose) breaks at seal on both web and CLI

**2. `cauldron run` breaks at seal stage**
- **Affected:** CLI-01
- `runCommand` calls `crystallizeCommand` which returns `seedId`, but doesn't extract it or inject `--seed-id` into seal args
- Seal exits with "Error: --seed-id is required"
- **Impact:** CLI full-pipeline command is non-functional

### P1 — Breaks Evolution Loop

**3. DB event vs Inngest event mismatch**
- **Affected:** HOLD-07, HOLD-08, EVOL-01 through EVOL-12
- `convergenceHandler` writes `evolution_started` to DB event store via `appendEvent()`
- `handleEvolutionStarted` listens for Inngest event `evolution_started`
- No bridge exists between DB events and Inngest events
- **Impact:** Evolution loop is unreachable from holdout evaluation

**4. Dispatch payload missing beadId**
- **Affected:** DAG-05, EXEC-03, EVOL-04
- `triggerExecution` tRPC and evolution dispatch send `bead.dispatch_requested` with `{seedId, projectId}` only
- `handleBeadDispatchRequested` expects `{beadId}` — receives undefined, produces no-op
- **Impact:** Web-initiated execution and evolution re-execution never dispatch beads

### P2 — Functional but Incomplete

**5. `bead_claimed` event never emitted** (WEB-03)
- `claimBead` in scheduler.ts never calls `appendEvent` → live DAG never shows `active` status

**6. Web SSE auth header gap** (WEB-04)
- `useSSE` uses browser `EventSource` which can't send `Authorization` headers
- When `CAULDRON_API_KEY` is set, web SSE connections silently fail with 401
