## Gaps Summary

One functional gap was found against the five must-have truths:

**Cross-model diversity not wired at gateway level.** The design requirement specifies Cousin Eddie runs on a configurable *different* model from the primary interviewer to achieve cross-model diversity. The config field (`contrarianModel`) and the pass-through spread were implemented, but the gateway's `generateObject` method unconditionally uses `resolveModelChain(options.stage)` to select models — it has no code path that reads `contrarianModel` or applies a per-call model override. As a result, Cousin Eddie always runs on the same `interview` stage model chain as the primary perspectives. The diversity goal is architecturally present (config field, type declaration, pass-through) but functionally inert (ignored at the routing layer).

The remaining four truths are fully satisfied: hypothesis framing, context injection, graceful failure, and backward compatibility are all working and tested. The gap is localized to a single missing feature in `gateway.generateObject`.
