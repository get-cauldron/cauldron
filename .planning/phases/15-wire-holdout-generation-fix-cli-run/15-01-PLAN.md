<objective>
Wire holdout scenario generation into the approveSummary tRPC mutation so the vault is populated after crystallization, and fix the CLI `cauldron run` command to propagate seedId from crystallize to seal stage.

Purpose: Close the two P0 integration gaps from the v1.0 milestone audit: (1) holdout generation is never triggered -- the engine code exists and is tested but nobody calls it, so the vault is always empty and sealing always fails; (2) `cauldron run` discards the seedId from crystallize, so the seal stage exits with "Error: --seed-id is required".

Output: Modified interview.ts (tRPC), crystallize.ts (CLI), run.ts (CLI)
</objective>

    2. Inside the `approveSummary` mutation, after the `crystallizeSeed` call succeeds and before the `return { seedId: seed.id, version: seed.version }` statement (around line 279), add holdout generation.

       The existing code structure is:
       ```typescript
       try {
         const seed = await crystallizeSeed(ctx.db, interview.id, projectId, summary, ambiguityScore);
         // INSERT HERE
         return { seedId: seed.id, version: seed.version };
       } catch (e) {
         if (e instanceof ImmutableSeedError) { ... }
         throw e;
       }
       ```

       Insert between `crystallizeSeed` and `return`:
       ```typescript
       // Generate holdout scenarios and create vault entry
       // Gateway diversity enforcement (LLM-06) is active for stage: 'holdout'
       try {
         const { gateway } = await ctx.getEngineDeps();
         const scenarios = await generateHoldoutScenarios({
           gateway,
           seed,
           projectId,
         });
         await createVault(ctx.db, { seedId: seed.id, scenarios });
       } catch (holdoutErr) {
         // Seed is already crystallized -- don't fail the mutation.
         // Log and continue; user can retry holdout generation separately.
         console.error('[approveSummary] Holdout generation failed:', holdoutErr);
       }
       ```

    IMPORTANT: The holdout generation try/catch is SEPARATE from the ImmutableSeedError catch. If holdout generation fails (LLM error, budget exceeded), the seed is already crystallized and should remain. The mutation still returns seedId. The user can retry holdout generation.

    Do NOT move getEngineDeps() outside the holdout try/catch -- keep it scoped so a getEngineDeps failure doesn't break the return.
