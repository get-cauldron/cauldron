<objective>
Two-pass LLM decomposition agent and DAG validation (cycle detection, size validation, coverage check).

Purpose: Implements the core intelligence of Phase 5 -- taking a crystallized seed and producing a valid, sized bead DAG. The decomposer calls the LLM twice (D-01): once for molecule hierarchy, once for atomic beads with edges. The validator ensures the DAG is acyclic (DAG-07), all beads fit in context windows (DAG-03), and every acceptance criterion is covered (D-09). Auto-retry on validation failure up to 3 attempts (D-04).

Output: `decomposer.ts` (two-pass LLM decomposition), `validator.ts` (Kahn's algorithm + size + coverage), and comprehensive unit tests.
</objective>

    Create `packages/engine/src/decomposition/validator.ts` with these exported functions:

    **`detectCycle(beadIds: string[], edges: Array<{ fromBeadId: string; toBeadId: string; edgeType: string }>): string[] | null`**
    - Implements Kahn's topological sort algorithm (BFS-based)
    - Filters edges to include `blocks`, `waits_for`, and `conditional_blocks` types only (exclude `parent_child` -- it tracks hierarchy, not scheduling order, per Research Pitfall 2)
    - Builds adjacency list and in-degree map from filtered edges
    - Queue starts with all zero-in-degree nodes
    - Process queue: decrement neighbors' in-degree, add newly zero-in-degree to queue
    - If `processed < beadIds.length`, return array of bead IDs with remaining in-degree > 0 (the cycle participants)
    - Return `null` if no cycle

    **`validateBeadSizes(beads: BeadSpec[], tokenBudget?: number): Array<{ beadId: string; estimatedTokens: number }>`**
    - Default tokenBudget = 200_000
    - Returns array of beads where `estimatedTokens > tokenBudget`
    - Empty array means all beads fit

    **`validateCoverage(beads: BeadSpec[], acceptanceCriteria: string[]): string[]`**
    - Collects all `coversCriteria` arrays from all beads into a Set
    - Returns acceptance criteria IDs NOT present in that Set
    - Empty array means full coverage

    **`validateDAG(result: DecompositionResult, acceptanceCriteria: string[], tokenBudget?: number): DAGValidationError | null`**
    - Builds edge array from beads' `dependsOn` (type: blocks), `waitsFor` (type: waits_for), `conditionalOn` (type: conditional_blocks)
    - Calls `detectCycle` -- if cycle found, return `{ type: 'cycle', message: 'Cycle detected involving beads: ${participants.join(", ")}', details: { cycleParticipants } }`
    - Calls `validateBeadSizes` -- if oversized, return `{ type: 'oversized_bead', message: '${count} bead(s) exceed ${budget} token limit: ${ids.join(", ")}', details: { oversizedBeads } }`
    - Calls `validateCoverage` -- if gaps, return `{ type: 'coverage_gap', message: 'Acceptance criteria not covered: ${uncovered.join(", ")}', details: { uncoveredCriteria } }`
    - Return `null` if all validations pass

    **`export async function decomposeSeed(options: { gateway: LLMGateway; seed: Seed; projectId: string; maxRetries?: number; tokenBudget?: number }): Promise<DecompositionResult>`**

    Implementation:
    1. Extract `acceptanceCriteria` from seed (it's a `string[]` in the JSONB column)
    2. Extract `ontologySchema` from seed (JSONB -- the entity map from Phase 3 D-24)
    3. Set `maxRetries = options.maxRetries ?? 3` and `tokenBudget = options.tokenBudget ?? 200_000`

    4. **Pass 1 -- Molecule decomposition:**
       - Call `gateway.generateObject({ projectId, stage: 'decomposition', schema: MoleculeOutputSchema, schemaName: 'MoleculeDecomposition', prompt: <see below>, system: <see below> })`
       - System prompt: Use the STAGE_PREAMBLES decomposition preamble (gateway handles this automatically)
       - User prompt: Include seed.goal, JSON.stringify(seed.constraints), JSON.stringify(seed.acceptanceCriteria), JSON.stringify(seed.ontologySchema). Ask LLM to produce molecule groupings where each molecule covers a logical subset of acceptance criteria.

    5. **Pass 2 -- Bead decomposition (with retry loop):**
       - `let lastError: DAGValidationError | null = null`
       - Loop up to `maxRetries` attempts:
         a. Build prompt with molecules from Pass 1, seed details, and tokenBudget. If `lastError`, append error-type-specific retry instructions:
            - **For `oversized_bead` errors (per D-07):** `"Previous attempt was invalid: ${lastError.message}. The following beads exceed the ${tokenBudget} token budget: ${JSON.stringify(lastError.details.oversizedBeads)}. Split each oversized bead into 2-3 smaller sub-beads. Each sub-bead must have its moleculeId set to the same parent molecule. Add 'blocks' dependency edges between sub-beads where ordering matters. Redistribute the original bead's coversCriteria across the sub-beads so no criteria are lost."`
            - **For `cycle` errors:** `"Previous attempt was invalid: ${lastError.message}. A dependency cycle was detected involving beads: ${lastError.details.cycleParticipants?.join(', ')}. Remove or reverse dependency edges to eliminate the cycle while preserving correct execution ordering."`
            - **For `coverage_gap` errors:** `"Previous attempt was invalid: ${lastError.message}. The following acceptance criteria are not covered by any bead: ${lastError.details.uncoveredCriteria?.join(', ')}. Add beads or update existing beads' coversCriteria to ensure every criterion is mapped to at least one bead."`
         b. Call `gateway.generateObject({ projectId, stage: 'decomposition', schema: DecompositionOutputSchema, schemaName: 'BeadDecomposition', prompt })`
         c. Construct `DecompositionResult` from Pass 1 molecules + Pass 2 beads
         d. Call `validateDAG(result, acceptanceCriteria as string[], tokenBudget)`
         e. If validation returns `null` -- return result (success)
         f. If validation returns error -- set `lastError = error`, continue loop
       - After loop exhaustion: throw `new Error('Decomposition failed after ${maxRetries} attempts: ${lastError!.message}')`
