## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip internal padding, badge padding |
| sm | 8px | Button internal padding (vertical), list item gaps, chip gaps |
| md | 16px | Default element spacing, sidebar item padding, card padding |
| lg | 24px | Section padding, panel header padding, card group gaps |
| xl | 32px | Layout column gaps, major section breaks within panels |
| 2xl | 48px | Page section breaks, between major layout zones |
| 3xl | 64px | Full-page padding (desktop only, 1280px+ target) |

Exceptions:
- Touch/click targets minimum 44px height (buttons, approval actions, MC chips) — enforced as minimum-height on interactive elements even though 44 is not a 4-multiple spacing token
- DAG node minimum width: 240px; minimum height: 80px
- Collapsible nav sidebar collapsed width: 56px; expanded width: 240px

**Source:** Standard 8-point scale. Touch target exception per accessibility standard. DAG/nav dimensions from CONTEXT.md D-10, D-25.

## Typography

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| Body | 14px | 400 | 1.5 | Geist Sans | Chat messages, descriptions, panel body text |
| Label | 12px | 500 | 1.4 | Geist Sans | Status badges, chip labels, metadata, nav items, dimension scores |

Display and Heading both weight 600 but different sizes — this satisfies the "2 weights" contract (regular 400, semibold 600; label uses 500 as a minor intermediate only for legibility at 12px).

### Accent Reserved-For (Teal `#00d4aa`)

Teal is reserved exclusively for:
1. Primary action buttons (Approve, Start Interview, Crystallize Seed, Confirm escalation response)
2. Hyperlinks and text links
3. Focus rings on all interactive elements
4. Active navigation item indicator (left border on sidebar)
5. Completed bead node borders and checkmark icons
6. Waits-for dependency edges in DAG (glowing teal per D-09)
7. Ambiguity score progress bar fill when score is positive
8. "Ready to crystallize" banner accent border

**Source:** CONTEXT.md D-29.

### Amber Reserved-For (Energy `#f5a623`)

Amber is reserved exclusively for:
1. Active bead node borders and glow
2. Progress bars and iteration count indicators on active beads
3. Convergence health indicators on timeline dots when stagnating
4. Cauldron logo/icon animated glow in nav header (D-32)
5. Escalation notification banners and badges (D-21)
6. Token cost trend indicators when over budget threshold
7. Hex background grid pulse near active elements (D-34)

**Source:** CONTEXT.md D-29.

## Animation Contract

All transitions use a consistent easing curve: `cubic-bezier(0.25, 0.46, 0.45, 0.94)` (ease-out-quad — feels mechanical and precise, not bouncy).

| Interaction | Duration | Treatment |
|-------------|----------|-----------|
| Bead status change glow pulse | 400ms | Opacity 0→1→0 on border glow; `ease-out-quad` |
| Dependency edge resolve flash | 600ms | Edge color transitions from muted to teal; `ease-out-quad` |
| Panel slide-in (bead detail) | 250ms | Translate from +100% to 0; `ease-out-quad` |
| Toast notification enter | 300ms | Translate from top -40px to 0 + fade in; `ease-out-quad` |
| Sidebar collapse/expand | 200ms | Width transition; `ease-out-quad` |
| MC chip selection | 150ms | Opacity fade-out on disappear; `ease-out-quad` |
| Hex background pulse near active | 2000ms | Opacity 0.03→0.08 on nearest hex cells; `ease-in-out`, looping |
| Camera auto-pan (DAG) | 800ms | React Flow `fitView` animation; `ease-out-quad` |
| Node hover | 150ms | Border opacity increase; `ease-out-quad` |

**Heavy effects excluded:** No particle effects, no morphing geometries, no 3D transforms.

**Source:** CONTEXT.md D-31 ("subtle glow transitions... no heavy particle effects"). Timing values are Claude's Discretion.

## Interaction States (required for every interactive element)

| State | Treatment |
|-------|-----------|
| Default | As specified in color contract |
| Hover | Surface-raised (`#1a2330`) background, teal border or underline |
| Focus | 2px teal (`#00d4aa`) outline, 2px offset |
| Active (pressed) | 10% opacity reduction on element |
| Disabled | 40% opacity, `cursor: not-allowed`, no hover effects |
| Loading | Skeleton shimmer or spinner in teal on dark background |
| Error | Red (`#e5484d`) border on input, error message below in red at 12px |

## Accessibility Baseline

- All interactive elements reachable via keyboard (Tab order, Enter/Space to activate)
- Focus rings always visible (never `outline: none` without replacement)
- ARIA labels on icon-only buttons and nav items
- Minimum contrast ratio 4.5:1 for body text (`#c8d6e5` on `#0a0f14` = 9.2:1, passes AAA)
- Color is never the sole differentiator for bead status — status icons accompany color
- Screen reader announcements for SSE status changes via `aria-live="polite"` regions
- DAG canvas provides text fallback list of beads when React Flow is not available
