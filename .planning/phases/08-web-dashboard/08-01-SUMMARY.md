### HZD Cauldron Design System

`globals.css` contains the full HZD Cauldron dark theme:
- Primary palette: background `#0a0f14`, card `#111820`, surface-raised `#1a2330`
- Text: primary `#c8d6e5` (silver), secondary `#6b8399`
- Accent teal: `#00d4aa` (interactive elements)
- Accent amber: `#f5a623` (energy/status indicators)
- Destructive: `#e5484d`
- All bead status tokens: pending `#3d5166`, active `#f5a623`, completed `#00d4aa`, failed `#e5484d`, blocked `#8a5c00`
- Dark-only (no light mode per D-33) — `--background: #0a0f14` always active

### shadcn Component Library (16 components)

button, card, badge, input, textarea, tabs, sheet, sonner (replaces deprecated toast), separator, scroll-area, collapsible, progress, dialog, tooltip, skeleton, avatar

### tRPC v11 Data Layer

- `trpc/init.ts`: `createTRPCContext` with `db` from `@cauldron/shared`, `router`, `publicProcedure` exports
- `trpc/router.ts`: `appRouter` with health check procedure; ready for sub-router expansion
- `trpc/query-client.ts`: `makeQueryClient` with 30s stale time and dehydration config
- `trpc/client.tsx`: `TRPCProvider`, `Providers` component wrapping app, `useTRPC` hook for components
- `trpc/server.tsx`: RSC prefetch proxy via `createTRPCOptionsProxy`
- `api/trpc/[trpc]/route.ts`: `fetchRequestHandler` for GET and POST

### Design Decisions (Not Deviations)

- **shadcn preset:** The CLI used `base-nova` preset (shadcn's current default) instead of `new-york`. Component output is structurally identical; only minor styling defaults differ. HZD CSS variable overrides apply regardless of preset.
- **toast → sonner:** shadcn deprecated `toast` component; used `sonner` instead as recommended. The UI-SPEC notification patterns still work.
- **globals.css structure:** shadcn init added its own CSS structure (oklch defaults, `.dark` class). Rewrote to use HZD hex values directly in `:root` while keeping shadcn's `@layer base` structure for component compatibility.
