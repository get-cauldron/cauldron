<objective>
Build the merge queue that serializes completed bead worktrees back to the main branch in DAG topological order, handles merge conflicts via LLM-assisted resolution with human escalation fallback, re-runs tests post-merge, and triggers knowledge graph re-indexing.

Purpose: Without the merge queue, parallel bead work cannot be integrated back. The topological ordering ensures upstream changes are available before downstream beads merge. Post-merge test re-runs catch cross-bead integration regressions.

Output: MergeQueue class with full merge lifecycle, conflict resolution, post-merge verification, and re-indexing triggers.
</objective>

    export type MergeStatus = 'merged' | 'conflict_resolved' | 'escalated' | 'reverted' | 'failed';

    `export class MergeQueue`:
      - `private queue: MergeQueueEntry[] = []`
      - Constructor: `constructor(private readonly worktreeManager: WorktreeManager, private readonly knowledgeGraph: KnowledgeGraphAdapter, private readonly gateway: LLMGateway, private readonly db: DbClient, private readonly projectRoot: string)`

      - `enqueue(entry: MergeQueueEntry): void`:
        - Push entry to queue
        - Sort queue by topologicalOrder ascending (D-15)

      - `async processNext(): Promise<MergeOutcome | null>`:
        - Shift first entry from sorted queue (lowest topologicalOrder)
        - If no entry, return null
        - Call `this.processMerge(entry)`

      - `async processAll(testRunner: TestRunnerConfig): Promise<MergeOutcome[]>`:
        - Process all entries sequentially in topological order
        - Return array of MergeOutcome

      - `private async processMerge(entry: MergeQueueEntry, testRunner: TestRunnerConfig): Promise<MergeOutcome>`:
        1. Attempt merge: `worktreeManager.mergeWorktreeToMain(entry.beadId, entry.branch)`
        2. If `result.conflicted`:
           a. Attempt LLM resolution: call `this.resolveConflict(entry, result.conflicts!)`
           b. If resolved (confidence high): commit resolution, retry merge step
           c. If not resolved (confidence low): abort merge (`git merge --abort`), emit `'merge_escalation_needed'` event via `appendEvent`, return `{ beadId, status: 'escalated', conflicts }`
        3. If merge succeeded:
           a. Run post-merge tests: execute `testRunner.unitCommand`, `testRunner.integrationCommand`, `testRunner.typecheckCommand` with `cwd: this.projectRoot`
           b. If tests pass:
              - Re-index knowledge graph: `this.knowledgeGraph.indexRepository()` (D-05, CODE-03)
              - Cleanup worktree: `this.worktreeManager.removeWorktree(entry.beadId)` (D-18)
              - Emit `'bead_merged'` event
              - Return `{ beadId, status: 'merged' }`
           c. If tests fail:
              - Revert merge: `git.raw(['reset', '--hard', 'HEAD~1'])` in projectRoot (D-16)
              - Emit `'merge_reverted'` event with test failure details
              - Keep worktree intact (D-18 — failed merges retain worktree)
              - Return `{ beadId, status: 'reverted', error: testFailureOutput }`

      - `private async resolveConflict(entry: MergeQueueEntry, conflicts: string[]): Promise<{ resolved: boolean; confidence: 'high' | 'low' }>`:
        - Read conflict markers from each conflicted file: `readFileSync(join(projectRoot, file), 'utf-8')`
        - Build prompt with both sides of conflict + bead spec context
        - Call `gateway.generateText({ stage: 'conflict_resolution', prompt, projectId: entry.projectId, beadId: entry.beadId })`
        - Parse response for confidence indicator (look for `"confidence": "low"` or `"confidence": "high"`)
        - If high confidence: write resolved files, `git add` each file, return `{ resolved: true, confidence: 'high' }`
        - If low confidence: `git merge --abort`, return `{ resolved: false, confidence: 'low' }`

      - `private async runPostMergeTests(testRunner: TestRunnerConfig): Promise<{ passed: boolean; errors: string[] }>`:
        - Run typecheck, unit, and integration commands sequentially with `cwd: this.projectRoot`
        - Return pass/fail status with error output
