<objective>
Build the worktree isolation layer and context assembly pipeline. WorktreeManager handles git worktree lifecycle (create, commit, cleanup). ContextAssembler queries the knowledge graph, scopes seed excerpts, and applies token budget trimming. TestDetector identifies the target project's test infrastructure.

Purpose: These two modules are consumed by the agent runner (Plan 03) and merge queue (Plan 04). Without worktree isolation, agents cannot execute in parallel. Without context assembly, agents get no useful code context.

Output: WorktreeManager class, ContextAssembler class, detectTestRunner function, simple-git dependency installed, unit tests for all three.
</objective>

    `export class ContextAssembler`:
      - `constructor(private readonly knowledgeGraph: KnowledgeGraphAdapter, private readonly gateway: LLMGateway)`
      - `async assemble(options: { bead: Bead; seed: Seed; projectId: string; projectRoot: string; dependencyOutputs?: string[] }): Promise<AgentContext>`:
        1. Extract keywords from bead spec and title via `extractKeywords(bead.spec, bead.title)`
        2. Query knowledge graph: `searchGraph({ name_pattern: keywords.join('|') })` — get candidate symbols
        3. For top symbols (limit 20), trace call paths: `traceCallPath(symbol.name, 'both')` — get 1-hop dependencies
        4. LLM pruning pass (D-07): call `gateway.generateObject({ stage: 'context_assembly', schema: PrunedSymbolsSchema, prompt: buildPruningPrompt(bead.spec, candidates), projectId })` to prune noise
        5. Fetch code snippets for pruned symbols: `knowledgeGraph.getCodeSnippet(qualifiedName)` for each
        6. Build seed excerpt via `buildSeedExcerpt(seed, bead.coversCriteria as string[])`: goal + all constraints + only matching acceptanceCriteria
        7. Build system prompt per D-11: define role as implementer, TDD approach (write tests first then implement), constraints (only modify files in worktree, no destructive operations), output format (code blocks with file paths)
        8. Build TestRunnerConfig via `detectTestRunner(projectRoot)`. If bead spec/title contain UI/API/CLI keywords, include e2eCommand; otherwise omit per D-23.
        9. Apply token budget via `applyTokenBudget()`: estimate tokens as `text.split(/\s+/).length * 1.3`, trim in D-10 priority order if over 180k

      - `private extractKeywords(spec: string, title: string): string[]`:
        - Split on whitespace and punctuation
        - Filter out common stop words (the, is, a, an, to, for, in, of, etc.)
        - Filter out words shorter than 3 chars
        - Deduplicate
        - Return top 15 keywords by frequency

      - `private buildSeedExcerpt(seed: Seed, coversCriteria: string[]): string`:
        - Start with `"## Goal\n{seed.goal}\n\n## Constraints\n"`
        - Add all constraints (cross-cutting per D-09)
        - Add `"## Acceptance Criteria (relevant to this bead)\n"`
        - Filter `seed.acceptanceCriteria` to only entries whose ID matches `coversCriteria`
        - If acceptanceCriteria is an array of strings, include those matching coversCriteria

      - `private buildSystemPrompt(): string`:
        - Role: "You are an implementation agent for Cauldron. You write production-quality TypeScript code."
        - TDD: "Write tests FIRST from the bead specification, then implement until all tests pass."
        - Constraints: "You may only create or modify files relative to the current working directory. No git push, no deletion outside scope, no network calls except through the provided LLM gateway."
        - Output format: "Respond with code blocks prefixed with the file path: ```typescript // path/to/file.ts"
        - Anti-mocking: "Prefer real integrations over mocks. Only mock true external services (third-party APIs, payment providers)."
        - Error handling: "If tests fail, read the error output carefully and modify your implementation. Do not modify tests to make them pass — fix the implementation."

      - `private applyTokenBudget(context: { seedExcerpt: string; beadSpec: string; codeSnippets: Array<{qualifiedName: string; code: string; filePath: string}>; dependencyOutputs: string[]; systemPrompt: string }, budget: number = 180_000): { trimmedContext: Omit<AgentContext, 'testRunner' | 'previousErrors'>; totalTokenEstimate: number }`:
        - Estimate each section's tokens: `words * 1.3`
        - If total <= budget, return as-is
        - Trim in priority order (D-10): (1) remove distant dependency code snippets (those with hop > 1), (2) truncate remaining code snippets to function signatures only, (3) truncate dependency outputs
