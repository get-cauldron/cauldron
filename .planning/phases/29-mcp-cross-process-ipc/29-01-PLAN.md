<objective>
Create the Redis pub/sub IPC modules: a publisher for the engine (Inngest worker) and a subscriber for the MCP stdio process.

Purpose: These are the two endpoints of the cross-process IPC bridge. The publisher fires after job state transitions in the Inngest worker; the subscriber receives messages in the MCP process and calls notifyJobStatusChanged. Both are error-tolerant -- Redis failures are logged, never thrown.

Output: ipc-publisher.ts, ipc-subscriber.ts, unit tests for both, ioredis added to MCP package.json
</objective>
