<objective>
Mount createInngestApp() on a running HTTP server so Inngest can deliver events to the 5 engine functions.

Purpose: Close the single blocking gap from VERIFICATION.md — createInngestApp() is defined but orphaned (never mounted on any HTTP server). The Inngest dev server in docker-compose.yml is already configured to poll http://host.docker.internal:3001/api/inngest, but nothing listens on port 3001. This plan creates the entry point that calls bootstrap(), creates the Inngest Hono app, and serves it via @hono/node-server on port 3001.

Output: A startable engine server at packages/api/src/engine-server.ts, a test, and docker-compose updated to also poll the web Inngest route on port 3000.
</objective>
