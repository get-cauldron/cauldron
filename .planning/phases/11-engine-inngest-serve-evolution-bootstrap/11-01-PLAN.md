  <action>
Create a new Next.js Route Handler at `packages/web/src/app/api/inngest/engine/route.ts` that serves the `cauldron-engine` Inngest client with all 5 engine functions.

CRITICAL: The engine functions require configured dependencies (configureSchedulerDeps, configureVaultDeps, configureEvolutionDeps) before they can handle events. The Next.js web app does NOT run bootstrap.ts — that is CLI-only. So this route must lazily initialize deps using the existing `getEngineDeps()` from `engine-deps.ts`.

Implementation:

```typescript
import { serve } from 'inngest/next';
import { db } from '@cauldron/shared';
import { getEngineDeps } from '../../../../trpc/engine-deps.js';
import {
  inngest as engineInngest,
  configureSchedulerDeps,
  configureVaultDeps,
  configureEvolutionDeps,
  handleBeadDispatchRequested,
  handleBeadCompleted,
  handleMergeRequested,
  handleEvolutionConverged,
  handleEvolutionStarted,
} from '@cauldron/engine';

// Lazy initialization — runs once per Node.js process.
// Next.js Route Handlers are server-side; module-level state persists across requests.
let initialized = false;
async function ensureInitialized(): Promise<void> {
  if (initialized) return;
  const { gateway } = await getEngineDeps();
  const projectRoot = process.env['CAULDRON_PROJECT_ROOT'] ?? process.cwd();
  configureSchedulerDeps({ db, gateway, projectRoot });
  configureVaultDeps({ db, gateway });
  configureEvolutionDeps({ db, gateway });
  initialized = true;
}

const inngestHandler = serve({
  client: engineInngest,
  functions: [
    handleBeadDispatchRequested,
    handleBeadCompleted,
    handleMergeRequested,
    handleEvolutionConverged,
    handleEvolutionStarted,
  ],
});

// Wrap each HTTP method to ensure deps are initialized before Inngest processes events
export async function GET(req: Request) {
  await ensureInitialized();
  return inngestHandler.GET(req);
}

export async function POST(req: Request) {
  await ensureInitialized();
  return inngestHandler.POST(req);
}

export async function PUT(req: Request) {
  await ensureInitialized();
  return inngestHandler.PUT(req);
}
```

Key decisions:
- Use `inngest as engineInngest` alias to avoid collision with `inngest` npm package name (same pattern as `execution.ts` router)
- Wrap GET/POST/PUT instead of exporting serve() directly so `ensureInitialized()` runs before each handler
- Use existing `getEngineDeps()` for gateway construction (already cached, already handles config loading)
- Import `db` from `@cauldron/shared` directly (same pattern as engine-deps.ts)
- `CAULDRON_PROJECT_ROOT` env var for project root (same as engine-deps.ts)
