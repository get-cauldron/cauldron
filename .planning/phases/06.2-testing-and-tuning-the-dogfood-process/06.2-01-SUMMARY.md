# Phase 06.2 Plan 01: Interview Convergence Bug Fixes Summary

Calibrated scorer prompt with explicit 0.8-1.0/0.4-0.7/0.0-0.3 anchors and recency-weighted prompt builder; dimension-aware perspective routing in mid/late turns targeting the weakest dimension specialist.

## Tasks Completed

### Task 1: Calibrate scorer system prompt with anchored examples and add recency weighting to prompt builder

- **Commit:** `7bde829`
- **Files:** `packages/engine/src/interview/scorer.ts`, `packages/engine/src/interview/__tests__/scorer.test.ts`

Rewrote `SCORER_SYSTEM_PROMPT` with explicit scoring anchors:
- `0.8-1.0`: specific, testable, unambiguous answers with concrete example
- `0.4-0.7`: partial clarity — intent expressed, details vague
- `0.0-0.3`: little or no information for the dimension
- Added MOST RECENT state of knowledge recency rule: later answers supersede earlier vagueness

Updated `buildScorerPrompt()` to split transcripts with 3+ turns into `EARLIER CONTEXT:` and `MOST RECENT ANSWERS (weight heavily):` sections. Short transcripts (1-2 turns) remain unsplit.

Added 7 regression tests covering:
- SCORER_SYSTEM_PROMPT contains `"0.8-1.0"`, `"specific, testable"`, `"MOST RECENT state of knowledge"`
- `buildScorerPrompt` with 5 turns contains `"MOST RECENT ANSWERS"`
- `buildScorerPrompt` with 3 turns (boundary) contains `"MOST RECENT ANSWERS"`
- `buildScorerPrompt` with 2 turns does NOT contain `"MOST RECENT ANSWERS"`
- `buildScorerPrompt` with 0 turns returns `"No interview turns yet"`

### Task 2: Add dimension-aware convergence pressure to perspective activation

- **Commit:** `eac3ae7`
- **Files:** `packages/engine/src/interview/perspectives.ts`, `packages/engine/src/interview/__tests__/perspectives.test.ts`

Enhanced `selectActivePerspectives()`:

**Mid-turn block (0.4 <= overall < 0.7):**
- Now evaluates all three dimensions against a 0.5 threshold
- Priority: `successCriteriaClarity < 0.5` → `seed-closer`; `constraintClarity < 0.5` → `breadth-keeper`; `goalClarity < 0.5` → `researcher`
- Fills remaining slots from `['breadth-keeper', 'simplifier']` avoiding duplicates
- If no dimension below 0.5: defaults to `['architect', 'breadth-keeper', 'simplifier']`

**Late-turn block (overall >= 0.7):**
- Adds a third specialist when any dimension is below 0.5
- `constraintClarity < 0.5` → adds `breadth-keeper`; `goalClarity < 0.5` → adds `researcher`; `successCriteriaClarity < 0.5` → adds `simplifier`
- Preserves `['seed-closer', 'architect']` 2-set when all dimensions are healthy

Added 5 regression tests:
- `mid-turn: low constraintClarity targets breadth-keeper`
- `mid-turn: low successCriteriaClarity targets seed-closer`
- `mid-turn: all dimensions above 0.5 defaults to architect+breadth-keeper+simplifier`
- `late-turn: low constraintClarity adds breadth-keeper as third perspective`
- `early-turn unchanged: overall=0.2 still returns researcher+simplifier+breadth-keeper`
