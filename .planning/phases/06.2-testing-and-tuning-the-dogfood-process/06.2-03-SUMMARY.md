## Decisions Made

- Zod `z.number()` without constraints is the only safe number type for LLM-facing schemas with Anthropic — minimum/maximum/integer type all rejected
- `z.record()` generates `propertyNames` which both Anthropic and OpenAI reject — use explicit `z.array(z.object({...}))` instead
- UUID validation moved to server-side post-processing — LLMs cannot be trusted to generate valid UUIDs in structured output
- `cauldron.config.ts` model IDs must reference real API-available models — speculative future models cause 500 errors

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Anthropic structured output rejects z.number().min(0).max(1) in scorer schema**
- **Found during:** Task 1 (interview stage - first LLM call)
- **Issue:** Anthropic API returns 400: "For 'number' type, properties maximum, minimum are not supported". z.number().min(0).max(1) generates `{"type": "number", "minimum": 0, "maximum": 1}` in JSON Schema — Anthropic rejects this.
- **Fix:** Changed `z.number().min(0).max(1)` to `z.number()` in greenfieldScoresSchema/brownfieldScoresSchema. Runtime [0,1] range validation already present in validateScoreRules (unchanged). Updated schema tests to reflect the change.
- **Files modified:** `packages/engine/src/interview/scorer.ts`, `packages/engine/src/interview/__tests__/scorer.test.ts`
- **Verification:** 44 interview tests pass. Interview ran successfully.
- **Committed in:** `2cc39e4`

**2. [Rule 1 - Bug] Anthropic rejects z.number().int() in ranker schema**
- **Found during:** Task 1 (ranker call after scoring)
- **Issue:** Anthropic API returns 400: "For 'integer' type, properties maximum, minimum are not supported". `z.number().int()` generates `{"type": "integer"}` — Anthropic rejects integer type.
- **Fix:** Changed to `z.number()`. Also removed `z.array().min(3).max(4)` (array minItems/maxItems).
- **Files modified:** `packages/engine/src/interview/ranker.ts`
- **Verification:** Interview converged successfully after fix.
- **Committed in:** `02879d1`

**3. [Rule 1 - Bug] Both Anthropic and OpenAI reject z.record() (propertyNames) in synthesizer**
- **Found during:** Task 1 (synthesis stage - after interview converged)
- **Issue:** Anthropic: "property 'propertyNames' is not supported". OpenAI: "'propertyNames' is not permitted". `z.record(z.string(), z.unknown())` generates `{"type": "object", "propertyNames": {...}}` in JSON Schema.
- **Fix:** Changed `exitConditions` from `z.record()` to `z.array(z.object({ condition: z.string(), description: z.string() }))`. Widened SeedSummary.exitConditions union type for backward compat.
- **Files modified:** `packages/engine/src/interview/synthesizer.ts`, `packages/engine/src/interview/types.ts`
- **Verification:** Seed summary generated and written successfully.
- **Committed in:** `02879d1`

**4. [Rule 1 - Bug] Google Gemini and OpenAI reject holdout schema constraints**
- **Found during:** Task 1 (seal stage - holdout generation)
- **Issue 1:** Google response didn't match schema — `z.string().uuid()` generates `{"type": "string", "format": "uuid"}` which LLMs may not satisfy. Also `z.array().min(5)` generates `minItems` which providers reject.
- **Issue 2:** cauldron.config.ts used non-existent model IDs `gemini-3.1-pro-preview` (future model, returns schema error), `gpt-5.4`, `gpt-5-mini`, `gpt-5-nano` (returns 500 errors).
- **Fix:** Changed `z.string().uuid()` to `z.string()`, removed `.min(5)`, added server-side UUID normalization. Updated config to `gemini-2.5-pro`, `gpt-4.1`, `gpt-4.1-mini`.
- **Files modified:** `packages/engine/src/holdout/types.ts`, `packages/engine/src/holdout/generator.ts`, `cauldron.config.ts`
- **Verification:** 7 holdout scenarios generated and sealed successfully.
- **Committed in:** `02879d1`

**5. [Rule 1 - Bug] Anthropic rejects z.number().int() in decomposer schema**
- **Found during:** Task 1 (decompose stage)
- **Issue:** `estimatedTokens: z.number().int()` generates integer type rejected by Anthropic. `coversCriteria: z.array().min(1)` generates minItems rejected by providers.
- **Fix:** Removed `.int()` and `.min(1)` from DecompositionOutputSchema.
- **Files modified:** `packages/engine/src/decomposition/decomposer.ts`
- **Verification:** Decomposition succeeded (13 beads created).
- **Committed in:** `4132c81`
