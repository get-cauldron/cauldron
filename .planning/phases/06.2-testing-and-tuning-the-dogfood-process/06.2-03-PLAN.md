    **Pre-flight checks:**
    1. Verify Docker services: `docker compose ps` -- postgres, redis, inngest must be running
    2. Verify health: `pnpm exec tsx packages/api/src/cli.ts health`
    3. Verify LLM API keys available: check `~/.env` for ANTHROPIC_API_KEY or OPENAI_API_KEY
    4. Verify Plan 01 fixes are in place: `grep "0.8-1.0" packages/engine/src/interview/scorer.ts` should match (scorer calibration applied)

    **Pipeline execution (sequential stages):**

    1. **Interview**: `pnpm exec tsx packages/api/src/cli.ts interview --project-root ~/Projects/cli-renamer-fixture`
       - Answer questions concretely (the CLI renamer is well-defined):
         - Goal: "A CLI tool that recursively renames files by replacing a pattern in filenames"
         - Constraints: "TypeScript, Unix/macOS, no external deps, must support dry-run"
         - Success criteria: "Accepts --dir, --find, --replace, --dry-run flags; handles edge cases"
       - Monitor the ambiguity score after each turn -- it should advance past 0.4 within 3-5 turns (indicating scorer calibration is working)
       - If the interview converges (score >= 0.8), proceed to reviewing state
       - **If score plateaus below 0.4 after 5+ concrete answers:** This means the scorer calibration from Plan 01 is insufficient. In this case:
         a. Read the scorer's actual LLM responses (enable debug logging or add a console.log in scoreTranscript)
         b. Identify what the scorer is undervaluing
         c. Adjust SCORER_SYSTEM_PROMPT with more explicit anchoring
         d. Re-run the interview from scratch (fresh transcript -- do NOT resume old session per RESEARCH.md Pitfall 7)
         e. Repeat up to 2 more times. If 3 total attempts fail, escalate: the scoring model may need structural redesign (D-05 fallback)

    2. **Crystallize**: Review the seed draft at `.cauldron/review/seed-draft-<projectId>.json`, then run:
       `pnpm exec tsx packages/api/src/cli.ts crystallize --project-id <projectId>`

    3. **Seal holdouts**: Generate and seal:
       `pnpm exec tsx packages/api/src/cli.ts seal --seed-id <seedId> --generate`
       Review draft, then: `pnpm exec tsx packages/api/src/cli.ts seal --seed-id <seedId>`

    4. **Decompose**: `pnpm exec tsx packages/api/src/cli.ts decompose --project-id <projectId> --seed-id <seedId>`

    5. **Execute**: `pnpm exec tsx packages/api/src/cli.ts execute --project-id <projectId> --seed-id <seedId>`

    **Diagnostic data to capture:**
    - After interview: final ambiguity scores (all dimensions + overall), number of turns, which perspectives were activated
    - After decompose: number of beads created, their titles
    - After execute: bead dispatch events in Inngest UI

    **If any stage fails**, diagnose and fix at source (per D-03). Any fix must include a regression test (per D-04). Common fixes:
    - Scorer still underscoring: further SCORER_SYSTEM_PROMPT calibration in scorer.ts
    - Perspective still stuck: threshold adjustment in perspectives.ts selectActivePerspectives
    - Decomposition fails: check seed content matches expected schema
    - Execute fails: check Inngest connection, API keys, project settings
