# Phase 6.2: Testing and Tuning the Dogfood Process - Context

**Gathered:** 2026-03-26
**Status:** Ready for planning

<domain>
## Phase Boundary

Run Cauldron's full pipeline end-to-end on an external CLI renamer project as a dry run, fixing whatever breaks along the way. Two parallel tracks: (1) fix interview convergence so it produces a crystallizable seed, and (2) exercise the downstream pipeline (holdout, decomposition, execution) using pre-generated seeds. Phase is complete when the full pipeline succeeds through at least one bead executing.

This is an iterative debugging/tuning phase, not a feature build. Scope emerges from what breaks during the dry run.

</domain>

<decisions>
## Implementation Decisions

### Dry-Run Target
- **D-01:** External CLI renamer project (separate repo, `selfBuild: false`). Low stakes, isolates pipeline bugs from self-referential complexity. Per Phase 6.1 D-09.

### Definition of Done
- **D-02:** Full pipeline through execute — interview converges, seed crystallizes, holdouts generated, decomposition succeeds, at least one bead executes. The entire chain must work end-to-end.

### Fix Strategy
- **D-03:** Fix engine code directly at the source (Phase 3/5/6 modules). These are bugs, not design changes.
- **D-04:** Every bug fix gets a regression test. Slower but prevents regressions from accumulating.

### Interview Convergence Approach
- **D-05:** Hybrid — start empirical (run interview, observe where it stalls, tweak thresholds, repeat). If 2-3 tuning passes don't produce convergence, step back and redesign the scoring model.
- **D-06:** Known convergence issues from dry-run: (a) perspective activation stuck on `researcher` — simplifier and seed-closer never activate, (b) scorer doesn't reward concrete answers — score oscillated 14-24% despite specific inputs, (c) no convergence pressure mechanism to shift focus from goal clarity to constraints/success criteria.

### Parallel Tracks
- **D-07:** Two parallel tracks to decouple problems:
  - **Track 1: Interview convergence** — Fix perspective rotation, scorer calibration, and convergence pressure in the Phase 3 interview engine.
  - **Track 2: Pre-generated seed pipeline** — Hand-craft a valid seed for the CLI renamer project, then exercise holdout generation → decomposition → execution independently. Unblocks downstream validation without waiting for interview fixes.

### Claude's Discretion
- Pre-generated seed content and structure (must conform to existing seed schema)
- CLI renamer fixture repo setup (minimal project structure)
- Order of convergence fixes within Track 1
- Regression test design and scope per fix
- Whether to parallelize tracks or sequence them based on complexity

</decisions>

- Interview engine modules are the primary fix targets (fsm.ts, perspectives.ts, scorer.ts)

<specifics>
## Specific Ideas

- Pre-generated seeds let us exercise holdout → decompose → execute without being blocked by interview convergence
- The CLI renamer is intentionally trivial — it should produce a small DAG (2-5 beads) that's easy to debug
- Interview convergence issues are well-documented from the dry-run: perspective stuck on researcher, score oscillation 14-24%, no convergence pressure
- Fix at source means modifying Phase 3 engine code directly, not adding workarounds in packages/api

</specifics>
