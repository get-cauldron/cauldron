## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed OpenAI structured output incompatibility in DecompositionOutputSchema**
- **Found during:** Task 2 pipeline execution (decompose stage)
- **Issue:** OpenAI gpt-4.1 strict mode requires ALL properties in the JSON Schema `required` array. The `conditionalOn: z.string().optional()` field was excluded from `required`, causing a 400 error: "In context=('properties', 'beads', 'items'), 'required' is required to be supplied and to be an array including every key in properties. Missing 'conditionalOn'."
- **Fix:** Changed `z.string().optional()` to `z.string().nullable()` in `DecompositionOutputSchema`. Updated `BeadSpec.conditionalOn` type from `string | undefined` to `string | null | undefined` for TypeScript compatibility. Existing `if (bead.conditionalOn)` guards remain correct for both `undefined` and `null`.
- **Files modified:** `packages/engine/src/decomposition/decomposer.ts`, `packages/engine/src/decomposition/types.ts`
- **Verification:** Decomposer unit tests (10/10 pass). Decompose command no longer throws 400 schema error — now progresses to LLM generation.
- **Committed in:** `3bae70e`
