    **1. Create `packages/cli/src/commands/rotate-kek.ts`:**

    Follow the `verify.ts` pattern: uses `parseArgs` from `node:util`, `chalk` for output, `bootstrap()` for direct DB access (no tRPC).

    Export: `async function rotateKekCommand(args: string[], flags: { json: boolean }): Promise<void>`

    The command has three modes, determined by args/flags:

    **Mode 1: `cauldron rotate-kek --init` (first-time setup)**
    - Parse `--init` flag, `--label` optional (default: `"v1-{ISO date}"`)
    - Read `HOLDOUT_ENCRYPTION_KEY` from env. If not set, error: "HOLDOUT_ENCRYPTION_KEY must be set"
    - Decode base64 to Buffer. Assert 32 bytes.
    - Call `initKekVersion(db, { kek, label })` from `@get-cauldron/engine`
    - Print: "KEK version {N} registered (fingerprint: {first 16 chars}...)"
    - If `--json` flag, output `{ version, fingerprint, label }` as JSON

    **Mode 2: `cauldron rotate-kek` (default — rotation)**
    - Parse `--new-key-env` flag (required, default: `'HOLDOUT_ENCRYPTION_KEY_NEW'`). This is the name of the env var holding the new key, NOT the key itself (keys must never appear in shell history).
    - Parse `--label` optional (default: `"v{N}-{ISO date}"`)
    - Read old key from `HOLDOUT_ENCRYPTION_KEY`, new key from the env var specified by `--new-key-env`. Both must be set.
    - Decode both from base64 to Buffer. Assert 32 bytes each.
    - Call `rotateKek(db, { oldKek, newKek, newKekLabel })` from `@get-cauldron/engine`
    - Print progress: "Rotating KEK: v{old} -> v{new}", "Re-encrypted {N} vault rows in {ms}ms"
    - Print reminder: "Set HOLDOUT_ENCRYPTION_KEY to the new key value and optionally set HOLDOUT_ENCRYPTION_KEY_PREV to the old key for dual-key window safety."
    - Print: "Run `cauldron rotate-kek --retire-old --old-version {old}` after confirming all in-flight evaluations are complete."
    - If `--json` flag, output the RotationResult as JSON

    **Mode 3: `cauldron rotate-kek --retire-old --old-version N`**
    - Parse `--retire-old` flag and `--old-version` (required integer)
    - Call `retireKek(db, { kekVersion })` from `@get-cauldron/engine`
    - Print: "KEK version {N} retired. You may now unset HOLDOUT_ENCRYPTION_KEY_PREV."
    - If `--json` flag, output `{ retired: true, version: N }` as JSON

    Error handling: wrap each mode in try/catch, print `chalk.red('Error: ...')` and `process.exit(1)`.

    Import `bootstrap` from `'../bootstrap.js'`. Import `rotateKek, retireKek, initKekVersion, kekFingerprint` from `'@get-cauldron/engine'` (the holdout module re-exports from rotation.ts).
