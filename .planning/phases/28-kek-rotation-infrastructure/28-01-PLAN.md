  <behavior>
    - reencryptDek: Given a compound encryptedDek string encrypted with oldKek, re-encrypts the DEK with newKek. Decrypting the result with newKek yields the original DEK bytes.
    - reencryptDek: Given an invalid format string (not 3 colon-separated parts), throws "Invalid encryptedDek format".
    - rotateKek: Given a 31-byte key, throws before touching any vault rows.
    - rotateKek: Given valid keys and vault rows with null encryptedDek, skips those rows and returns rowsRotated=0 for them.
    - rotateKek: Given valid keys and N sealed vault rows, re-encrypts all N and returns rowsRotated=N.
    - rotateKek: Inserts a new row into kek_versions with status='active' and correct fingerprint.
    - rotateKek: Inserts 2 rows into kek_rotation_log (rotation_started, rotation_completed).
    - retireKek: Given a version with vault rows still referencing it, throws "Cannot retire".
    - retireKek: Given a version with zero referencing vault rows, sets status='retired' and retiredAt, inserts kek_rotation_log 'old_key_retired'.
    - kekFingerprint: SHA-256 hex digest of the key buffer.
  </behavior>

    Implement the following exports:

    `kekFingerprint(kek: Buffer): string` — SHA-256 hex digest of key buffer.

    `reencryptDek(encryptedDek: string, oldKek: Buffer, newKek: Buffer): string` — Parses the compound DEK string (dekIv:dekAuthTag:dekCiphertext, all base64), decrypts with oldKek using AES-256-GCM, re-encrypts with newKek using fresh random 12-byte IV, returns new compound string in same format. Throws on invalid format (not 3 parts) or auth tag mismatch.

    `rotateKek(db: DbClient, params: { oldKek: Buffer; newKek: Buffer; newKekLabel: string }): Promise<RotationResult>` where `RotationResult = { rowsRotated: number; newKekVersion: number; oldKekVersion: number }`:
    1. Assert both keys are exactly 32 bytes. Throw before any DB operation if not.
    2. Find current active KEK version from kek_versions where status='active'. Throw if none found.
    3. Verify oldKek fingerprint matches the active version's key_fingerprint. Throw if mismatch (prevents rotating with wrong old key).
    4. Insert new kek_versions row with status='active', the newKekLabel, and SHA-256 fingerprint of newKek.
    5. Insert kek_rotation_log row: event='rotation_started', payload={oldKekVersion, newKekVersion, totalRows}.
    6. Query holdoutVault rows where encryptedDek IS NOT NULL (use `isNotNull` from drizzle-orm).
    7. For each row, call reencryptDek, then update that row's encryptedDek and kekVersion in a transaction.
    8. Insert kek_rotation_log row: event='rotation_completed', payload={rowsRotated, newKekVersion, durationMs}.
    9. Return RotationResult.

    `retireKek(db: DbClient, params: { kekVersion: number }): Promise<void>`:
    1. Query holdoutVault for any rows where kekVersion equals params.kekVersion. If any exist, throw "Cannot retire KEK v{N}: vault rows still reference it".
    2. Update kek_versions row: set status='retired', retiredAt=new Date().
    3. Insert kek_rotation_log row: event='old_key_retired', payload={retiredKekVersion, retiredAt}.

    `initKekVersion(db: DbClient, params: { kek: Buffer; label: string }): Promise<number>`:
    1. Insert a kek_versions row with status='active' and the SHA-256 fingerprint.
    2. Return the version number. This is used for first-time setup before any rotation.

    Import `createCipheriv, createDecipheriv, createHash, randomBytes` from 'node:crypto'.
    Import `eq, isNotNull` from 'drizzle-orm'.

    Use constants: `ALGORITHM = 'aes-256-gcm'`, `IV_LENGTH = 12`, `KEY_LENGTH = 32`.

    **1. Update `packages/engine/src/holdout/crypto.ts`:**

    Add a new exported function `unsealPayloadWithFallback` that:
    1. First tries `unsealPayload(sealed)` (uses current HOLDOUT_ENCRYPTION_KEY).
    2. On any error, checks if `process.env['HOLDOUT_ENCRYPTION_KEY_PREV']` is set.
    3. If set, creates a Buffer from it (base64), then manually performs the same decrypt logic as `unsealPayload` but using the prev KEK buffer instead of calling `getKek()`.
    4. If HOLDOUT_ENCRYPTION_KEY_PREV is not set, throws: "Decryption failed and no HOLDOUT_ENCRYPTION_KEY_PREV available for fallback".

    To avoid duplicating the decrypt logic, refactor by extracting a private helper `unsealPayloadWithKek(sealed: SealedPayload, kek: Buffer): string` that does the actual decryption given an explicit KEK. Then:
    - `unsealPayload` calls `unsealPayloadWithKek(sealed, getKek())`
    - `unsealPayloadWithFallback` tries current KEK, catches, tries prev KEK

    Export `unsealPayloadWithFallback` from the module. Do NOT change the existing `unsealPayload` signature or behavior.
