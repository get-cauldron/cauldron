Holdout evaluator, unseal protocol, and Inngest event-driven convergence handler.

Purpose: Completes the holdout vault lifecycle by implementing unsealing (HOLD-06), LLM-based evaluation of holdout scenarios (HOLD-07), and failure-triggered re-evolution (HOLD-08). The Inngest event handler wires these together: on evolution_converged, unseal the vault, evaluate scenarios, and if any fail, emit an evolution_started event with the failure context for Phase 7's evo loop.
