    2. `createVault(db: DbClient, params: { seedId: string; scenarios: HoldoutScenario[] }): Promise<string>`:
       - Insert into `holdoutVault` with `status: 'pending_review'`, `draftScenarios: scenarios` as JSONB
       - All encryption columns (`ciphertext`, `encryptedDek`, `iv`, `authTag`, `encryptedAt`) are null

    3. `approveScenarios(db: DbClient, params: { vaultId: string; approvedIds: string[] | 'all' }): Promise<{ approved: number }>`:
       - Read current vault row
       - If `approvedIds === 'all'`: approve all scenarios in `draftScenarios`
       - Otherwise: mark the specified scenario IDs as approved in the draft
       - Count approved. If fewer than 5, throw `new Error('Minimum 5 approved scenarios required to proceed. Currently approved: {n}')`
       - Update vault status to `'approved'`, update `draftScenarios` with approval metadata
       - Validate transition via `VALID_TRANSITIONS`

    4. `rejectScenarios(db: DbClient, params: { vaultId: string; rejectedIds: string[]; reasons: string[] }): Promise<{ rejectedIds: string[]; reasons: string[] }>`:
       - Does NOT change vault status (stays pending_review for regeneration)
       - Returns the rejected IDs and reasons for use with `regenerateRejected()`

       - Call `appendEvent(db, { projectId, seedId: vault.seedId, type: 'holdouts_sealed', payload: { vaultId, scenarioCount } })`
