    **Create packages/engine/src/holdout/crypto.ts** implementing per D-10 envelope encryption:

    1. Constants: `ALGORITHM = 'aes-256-gcm'`, `IV_LENGTH = 12` (96-bit, GCM standard), `KEY_LENGTH = 32` (256-bit).

    2. `getKek(): Buffer` — reads `process.env['HOLDOUT_ENCRYPTION_KEY']`, throws `new Error('HOLDOUT_ENCRYPTION_KEY is required for vault operations')` if absent/empty. Decodes from base64. NEVER exported — internal only. NEVER log or return the key.

    3. `sealPayload(plaintext: string): SealedPayload` — per D-10:
       - Call `getKek()` to get KEK
       - Generate random DEK: `randomBytes(KEY_LENGTH)` (32 bytes)
       - Generate random IV: `randomBytes(IV_LENGTH)` (12 bytes)
       - Encrypt plaintext with DEK using AES-256-GCM: `createCipheriv(ALGORITHM, dek, iv)`
       - Get auth tag: `cipher.getAuthTag()`
       - Encrypt DEK with KEK using separate AES-256-GCM: fresh `dekIv = randomBytes(IV_LENGTH)`, fresh cipher
       - Store encrypted DEK as compound string: `${dekIv.toString('base64')}:${dekAuthTag.toString('base64')}:${encDek.toString('base64')}`
       - Return `{ ciphertext, iv, authTag, encryptedDek }` all as base64 strings
       - Import `SealedPayload` from `'./types.js'`

    4. `unsealPayload(sealed: SealedPayload): string` — reverse of sealPayload:
       - Call `getKek()` to get KEK
       - Parse `encryptedDek` compound string by splitting on ':'
       - Decrypt DEK with KEK
       - Decrypt ciphertext with DEK
       - Return plaintext string

    5. Import from `'node:crypto'`: `createCipheriv`, `createDecipheriv`, `randomBytes`.
    6. Use `.js` extension on relative imports.
