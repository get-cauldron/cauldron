    **0. Add triggerDecomposition and triggerExecution mutations to execution router** (fixes blocker: decompose/execute commands need tRPC procedures for async Inngest triggers):

    In `packages/web/src/trpc/routers/execution.ts`, add two new mutations:

    ```typescript
    // Trigger decomposition (emits Inngest event for async processing)
    triggerDecomposition: publicProcedure
      .input(z.object({
        projectId: z.string().uuid(),
        seedId: z.string().uuid(),
      }))
      .mutation(async ({ ctx, input }) => {
        await appendEvent(ctx.db, {
          projectId: input.projectId,
          beadId: null,
          type: 'decomposition_started',
          payload: { seedId: input.seedId, source: 'cli' },
        });
        return { success: true, message: 'Decomposition triggered' };
      }),

    // Trigger execution (emits Inngest event for async processing)
    triggerExecution: publicProcedure
      .input(z.object({
        projectId: z.string().uuid(),
        seedId: z.string().uuid(),
      }))
      .mutation(async ({ ctx, input }) => {
        await appendEvent(ctx.db, {
          projectId: input.projectId,
          beadId: null,
          type: 'execution_started',
          payload: { seedId: input.seedId, source: 'cli' },
        });
        return { success: true, message: 'Execution triggered' };
      }),
    ```

    NOTE: The event types (`decomposition_started`, `execution_started`) must exist in the eventTypeEnum. Check `packages/shared/src/schema.ts` -- if they don't exist, add them via migration. The Inngest function that listens for these events to actually run decomposition/execution is in the engine package and should already respond to these event types.
