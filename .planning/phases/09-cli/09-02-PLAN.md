From projects router:
- client.projects.list.query() → { id, name, description, lastActivity, lastEventType, totalCostCents }[]
- client.projects.create.mutate({ name, description? }) → project
- client.projects.archive.mutate({ id }) → { success: true }
- client.projects.byId.query({ id }) → project with totalCostCents

From interview router:
- client.interview.getTranscript.query({ projectId }) → { transcript, currentScores, status, phase, suggestions, activePerspective, thresholdMet }
- client.interview.sendAnswer.mutate({ projectId, answer, freeformText? }) → { interviewId, turnNumber, currentScores, thresholdMet, phase }
- client.interview.getSummary.query({ projectId }) → { summary, phase, interviewId }
- client.interview.approveSummary.mutate({ projectId, summary }) → { seedId, version }
- client.interview.rejectSummary.mutate({ projectId }) → { interviewId, phase }
- client.interview.getHoldouts.query({ seedId }) → { scenarios[] }
- client.interview.approveHoldout.mutate({ holdoutId }) → { holdoutId, status }
- client.interview.sealHoldouts.mutate({ seedId }) → { seedId, sealedCount }

From execution router:
- client.execution.getProjectDAG.query({ projectId }) → { beads[], edges[], seedId }
- client.execution.getBeadDetail.query({ beadId }) → { bead, events[] }
- client.execution.respondToEscalation.mutate({ projectId, beadId?, action, guidance? })

From costs router:
- client.costs.getProjectSummary.query({ projectId }) → { totalCostCents, totalPromptTokens, totalCompletionTokens, totalTokens, callCount }
- client.costs.getByModel.query({ projectId }) → { model, totalCostCents, totalTokens, callCount }[]
- client.costs.getByStage.query({ projectId }) → { stage, totalCostCents, totalTokens, callCount }[]

From evolution router:
- client.evolution.getSeedLineage.query({ projectId }) → seed[]
- client.evolution.getEvolutionHistory.query({ projectId }) → event[]
- client.evolution.getConvergenceForSeed.query({ seedId }) → { convergenceEvent, lateralThinkingEvents, costCents, totalTokens }

    **interview.ts** (per D-06):
    - Interactive terminal mode (default): Use readline/process.stdin to display numbered MC options + freeform input
    - Loop: call `client.interview.getTranscript.query({ projectId })` to get current state
    - Display question, MC options (numbered), freeform option
    - Read user input, call `client.interview.sendAnswer.mutate({ projectId, answer })`
    - Show spinner while waiting, then display updated scores
    - When thresholdMet === true, show summary via `client.interview.getSummary.query({ projectId })`
    - Prompt user to approve/reject summary
    - `--browser` flag: open dashboard URL in browser via `import('open')` or `child_process exec 'open URL'`

    **crystallize.ts**:
    - Get summary via `client.interview.getSummary.query({ projectId })`
    - If phase is reviewing, call `client.interview.approveSummary.mutate({ projectId, summary })`
    - Print "Seed crystallized: {seedId}"

    **seal.ts**:
    - Get holdouts via `client.interview.getHoldouts.query({ seedId })`
    - Display scenarios for review
    - Approve each via `client.interview.approveHoldout.mutate({ holdoutId })`
    - Seal via `client.interview.sealHoldouts.mutate({ seedId })`

    **decompose.ts**:
    - This requires triggering decomposition which runs asynchronously.
    - The tRPC router does not currently have a `decompose` mutation. Since the decomposition runs via Inngest events (not a direct function call), the CLI should emit the event or call a new tRPC procedure.
    - For now, add a note that the tRPC router may need a `triggerDecomposition` mutation added. If the procedure is not available, keep the existing engine-direct call as a fallback with a TODO comment for adding the tRPC procedure.

    **execute.ts**:
    - Similar to decompose — execution dispatch runs via Inngest events.
    - If tRPC procedure is unavailable, keep engine-direct call with TODO.

    **status.ts**:
    - Replace direct DB queries with `client.execution.getProjectDAG.query({ projectId })`
    - Render bead table using createTable() and colorStatus()
    - Show summary line: X completed, Y active, Z pending, W failed
    - Support --logs flag by tailing events via `client.execution.getBeadDetail.query()`

    **kill.ts**:
    - Call `client.execution.respondToEscalation.mutate({ projectId, action: 'abort' })`
    - Print "Pipeline killed for project {projectId}"

    **resolve.ts**:
    - Accept bead ID and action (retry/skip/guidance)
    - Call `client.execution.respondToEscalation.mutate({ projectId, beadId, action, guidance })`

    IMPORTANT: Do NOT import from @cauldron/engine or @cauldron/shared in refactored commands. The anti-pattern from research: "After the tRPC migration, CLI commands must not bootstrap engine dependencies."
    EXCEPTION: decompose.ts and execute.ts may temporarily retain engine imports if tRPC procedures for triggering async Inngest events don't exist yet. Mark with `// TODO(09): Add tRPC procedure for async trigger` comments.
