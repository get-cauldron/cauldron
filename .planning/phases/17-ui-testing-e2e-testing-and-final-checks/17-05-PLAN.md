    async function globalSetup() {

    **TypeScript strict audit (per D-18):**

    Run `grep -rn ': any\|as any\|<any>\| any;' packages/*/src/ --include='*.ts' --include='*.tsx'` to identify all `any` usages across the codebase.

    For each `any` found, categorize as:
    1. **SDK boundary escape hatch** — forced by upstream SDK type issues (AI SDK v6 `Promise<any>`, Inngest generics `InngestFunction<any>`). These are documented in STATE.md. Leave them but ensure each has a `// eslint-disable-next-line @typescript-eslint/no-explicit-any -- {reason}` comment or equivalent JSDoc explaining why.
    2. **Sloppy typing** — `any` used where a proper type exists. Fix these by adding the correct type.
    3. **Logger boundary** — `logger:any` at engine-deps bridge (documented in Phase 10 decisions). Leave with comment.

    For sloppy `any` usages found: replace with proper types. For boundary escape hatches: add justification comment if missing.

    The goal per D-18 is zero *unexplained* `any` — every `any` in the codebase must have a comment explaining why it cannot be replaced (per research pitfall 7).

    **Dependency vulnerability audit (per D-19):**
    Run `pnpm audit --audit-level high`. If any critical/high vulnerabilities are found, fix them by updating the affected dependency version.

    **License compliance (per D-19):**
