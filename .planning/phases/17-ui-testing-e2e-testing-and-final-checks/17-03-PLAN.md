**D-05 Architectural Note:** Investigation of the codebase confirms that ALL LLM calls are server-side. The call path is: Browser -> tRPC mutation (HTTP POST) -> Next.js server -> InterviewFSM.submitAnswer() -> gateway -> AI SDK -> Anthropic API. Playwright's `page.route()` only intercepts browser-originating HTTP requests and CANNOT intercept server-to-server calls from the Next.js process. Therefore, D-05 is achieved by pre-seeding interview transcript data directly in the database, which avoids hitting real LLM providers entirely. The interview E2E tests verify rendering of pre-seeded transcripts and user input submission (which triggers tRPC calls), but do not test the full LLM response round-trip -- that is covered by integration tests in the engine package.
</objective>

    **D-05 Mock Strategy -- Pre-seeded Data (NOT page.route()):**

    Investigation confirmed that ALL AI SDK calls are server-side. The flow is:
    `Browser tRPC POST -> Next.js server -> InterviewFSM.submitAnswer() -> gateway -> AI SDK -> api.anthropic.com`

    Playwright `page.route()` only intercepts browser-originating HTTP. Server-to-server calls from the Next.js process are invisible. Therefore, D-05 (LLM mocking) is achieved by:

    1. **Pre-seeding interview transcript data** directly into the `interviews` table with realistic multi-turn conversations, ambiguity scores, MC options, and phase transitions. The `getTranscript` tRPC query reads directly from DB -- no LLM call needed for rendering.
    2. **Testing user input submission** (typing + send) triggers the tRPC `sendAnswer` mutation. The E2E test verifies the mutation fires (the input clears, "Sending..." state shows) but does NOT wait for an LLM response. This tests the UI wiring without requiring a live LLM.
    3. **No real LLM provider is ever called** because all test scenarios use pre-existing DB state.

    This approach is more robust than `page.route()` interception because it tests the actual data path (DB -> tRPC -> UI) that production uses for rendering.

    **Setup/teardown:** `test.beforeAll` -- create DB via `createE2EDb()`, run migrations. `test.afterEach` -- `truncateE2EDb(db)`. `test.afterAll` -- close client.
