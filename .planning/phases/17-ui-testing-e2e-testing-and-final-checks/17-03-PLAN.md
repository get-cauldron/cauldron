    **LLM route interception (per D-05):**
    In `test.beforeEach`, set up `page.route()` to intercept:
    - `https://api.anthropic.com/**` — return SSE-formatted mock responses from `interview-responses.json` fixture
    - `https://api.openai.com/**` — return JSON mock response
    - `https://generativelanguage.googleapis.com/**` — return JSON mock response

    The fixture format for Anthropic streaming must match the SSE event stream format:
    ```
    data: {"type":"content_block_start","index":0,"content_block":{"type":"text","text":""}}
    data: {"type":"content_block_delta","index":0,"delta":{"type":"text_delta","text":"..."}}
    data: {"type":"message_stop"}
    ```

    For `generateObject` calls (scorer, summary), the AI SDK expects a JSON response body, not SSE.

    **Setup/teardown:** `test.beforeAll` — create DB, run migrations. `test.afterEach` — truncate. `test.afterAll` — close client.

    **Tests:**
    1. `test('interview page loads with initial question')` — seed project + interview with one assistant message in transcript, navigate to interview page, assert chat bubble with assistant text visible, `assertNoA11yViolations(page)`, `expect(page).toHaveScreenshot('interview-initial.png', { threshold: 0.1, animations: 'disabled' })`

    2. `test('user can send an answer via input')` — seed project + active interview, navigate to interview page, locate textarea input, type "I want to build a CLI file renamer", click send button, assert user message bubble appears with typed text. Wait for LLM mock to resolve (the intercepted route will fire), then assert a new assistant response appears.

    3. `test('MC chips are clickable and submit answer')` — seed project + interview with MC suggestions in transcript (add to factory), navigate, assert MC chip buttons rendered, click one, assert it submits as the answer.

    4. `test('ambiguity meter updates after response')` — seed project + interview with scores in DB, navigate, assert ambiguity meter component visible with dimension scores.

    5. `test('interview page has no accessibility violations')` — seed project + interview, navigate, `assertNoA11yViolations(page)`.

    **SSE real event test (per D-15):**
    6. `test('SSE delivers real-time events to UI')` — seed project, navigate to interview page (which opens SSE connection), insert a new event row directly into the `events` table via `createTestEvent(db, projectId, 'interview_updated', { ... })`, then wait with `expect(locator).toBeVisible({ timeout: 8000 })` for the UI to reflect the update (covering the 2s polling interval).

    **Avoid `page.waitForLoadState('networkidle')`** — SSE holds the connection open, so networkidle never fires. Use element-based waits.

    **Important:** Read the interview page source to understand the exact tRPC calls made, the exact DOM structure (button selectors, input selectors), and how the page fetches/displays interview data. The mock route interception only works if it matches the URLs the AI SDK actually calls from the server side. If the page uses server-side tRPC calls (which call the engine, which calls AI SDK), the route interception must happen at the provider API URL level, not the tRPC level.

    **If the interview flow requires Inngest to be running (for question generation), scope the E2E test to pre-seeded transcript data and test the rendering + user input submission, not the full LLM response cycle.** The route interception approach works for client-side AI SDK calls but may not intercept server-to-server calls. Adapt accordingly.
