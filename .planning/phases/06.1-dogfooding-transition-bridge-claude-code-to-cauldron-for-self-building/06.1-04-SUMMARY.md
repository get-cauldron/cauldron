# Phase 6.1 Plan 04: Execution Pipeline Commands and Self-Build Safety ŌĆö Summary

Implemented the full execution pipeline CLI commands (decompose, execute, seal, resolve) and self-build safety mechanisms (engine snapshot, migration detection), completing the dogfooding transition CLI surface.

## What Was Built

### Task 1: Decompose, Seal, and Resolve Commands

**decompose.ts** ŌĆö Drives seed decomposition pipeline:
- Parses `--project-id`, `--seed-id`, `--project-root` from process.argv
- Queries most recent crystallized seed if `--seed-id` not provided
- Calls `runDecomposition({ db, gateway, inngest, seed, projectId, tokenBudget: 180_000 })`
- Prints bead count with follow-up command hint

**seal.ts** ŌĆö Two-mode holdout vault lifecycle:
- `--generate` mode: calls `generateHoldoutScenarios`, `createVault`, `writeHoldoutDraft`, exits prompting review
- Seal mode (no flag): reads draft, filters approved scenarios, calls `approveScenarios` + `sealVault`
- Guards against zero approved scenarios (exits 1 with message)

**resolve.ts** ŌĆö Merge conflict resolution:
- Checks for conflict diff file at `{projectRoot}/.cauldron/review/conflict-{beadId}.diff`
- Looks up projectId via seed join (beads don't store projectId directly)
- Resets bead status from `failed` ŌåÆ `pending`
- Appends `conflict_resolved` event via appendEvent

**Deviation (Rule 2 ŌĆö Missing Critical Functionality):** Added `conflict_resolved` to `eventTypeEnum` in `packages/shared/src/db/schema/event.ts` and created migration `0008_conflict_resolved_event.sql`. This was required for the resolve command's `appendEvent` call to typecheck correctly. Without it, TypeScript rejected the event type string.

### Task 2: Execute Command and Self-Build Safety

**execute.ts** ŌĆö Inngest HTTP server + bead dispatch:
- Starts Hono server on port 3001 with `inngestServe({ client, functions })`
- Functions array: `handleBeadDispatchRequested`, `handleBeadCompleted`, `handleMergeRequested`, `handleEvolutionConverged`
- `--resume` flag: resets `failed` beads to `pending` before finding ready beads
- Dispatches each ready bead via `inngest.send({ name: 'bead.dispatch_requested', data: { beadId, seedId, projectId } })`
- When `config.selfBuild === true`: captures engine snapshot and registers SIGINT handler to warn on engine change

**self-build.ts** ŌĆö Engine safety utilities:
- `captureEngineSnapshot(projectRoot)`: runs `git rev-parse HEAD` via execSync, returns 40-char hash
- `detectEngineChange(snapshotHash, projectRoot)`: compares current HEAD to snapshot
- `hasMigrationFiles(diffOutput)`: checks if diff contains `packages/shared/src/db/migrations/`
- `getBeadDiff(projectRoot, beadBranch)`: `git diff main...{branch} --name-only` for migration checking

**Deviation (Rule 2 ŌĆö Missing Critical Functionality):** Exported `handleMergeRequested` from `packages/engine/src/decomposition/index.ts`. It was defined in `events.ts` but not re-exported via the index, making it inaccessible to the execute command via `@cauldron/engine`.
