### Auto-fixed Issues

**1. [Rule 1 - Bug] LLMGateway mock in bootstrap.test.ts required function-constructor syntax**
- **Found during:** Task 2 GREEN phase
- **Issue:** The test used `vi.fn().mockImplementationOnce(() => mockGateway)` which creates an arrow function, but `new LLMGateway(...)` requires a callable constructor. Vitest throws "is not a constructor" at runtime.
- **Fix:** Changed mock to `vi.fn(function () { return { streamText: vi.fn() }; })` â€” a named function is callable as a constructor.
- **Files modified:** `packages/api/src/__tests__/bootstrap.test.ts`
- **Commit:** 5a88853 (bundled with GREEN implementation)
