# Phase 6.1 Plan 02: Status, Kill, and Review Writers Summary

Status/kill commands and seed/holdout review writers implemented with TDD — enabling observability, stuck-bead intervention, and file-based review workflows for the interview and seal pipeline stages.

## Completed Tasks

### Task 1: Status command and kill command (TDD)

**RED:** 5 failing tests committed — statusCommand table render, --logs flag, NEEDS REVIEW detection, killCommand success and bead-not-found paths.

**GREEN:** Implemented:
- `status.ts`: queries beads by seedId (or most recent seed if omitted), computes duration from claimedAt/completedAt timestamps, queries `merge_escalation_needed` events and overrides status display to "NEEDS REVIEW" for matching beads, renders with `console.table()`, prints completion summary line, tails last 20 events when `--logs` flag present
- `kill.ts`: verifies bead exists, updates status to `'failed'`, appends `bead_failed` event via `appendEvent`, prints confirmation; exits 1 on missing bead
- `cli.ts`: updated to call `bootstrap()` and pass deps + commandArgs to status/kill; other stub commands unchanged

All 5 tests pass.
