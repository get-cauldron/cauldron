## Decisions Made

- Skill files are plain markdown — no manifest or registration needed; Claude Code reads them directly when a skill is invoked
- Health check pre-flight (`pnpm exec tsx packages/api/src/cli.ts health`) included in every skill to prevent silent failures when Docker services are down
- Inngest v4 health endpoint is `/v1/events` (not `/v0/envs`) — corrected during Task 1 execution

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed Inngest health check endpoint returning 404**
- **Found during:** Task 1 (Create all 8 Claude Code skill files) — health check verification step
- **Issue:** `packages/api/src/commands/health.ts` was calling `/v0/envs` which returns 404 in Inngest v4; health command was incorrectly reporting the Inngest service as unhealthy even when running
- **Fix:** Updated Inngest health probe to POST to `/v1/events` with an empty events array, which returns 200 when the dev server is up
- **Files modified:** `packages/api/src/commands/health.ts`
- **Verification:** `pnpm exec tsx packages/api/src/cli.ts health` returned "All services healthy"
- **Committed in:** `66cbbfd` (fix(06.1-05))
