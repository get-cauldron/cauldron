    **Step 1 — Create `packages/cli/src/commands/verify.ts` (per D-06, D-09):**

    This command uses `bootstrap()` directly (like `health.ts`), NOT `bootstrapClient()`. It needs direct DB access and engine wiring, not tRPC. Require `--project` flag.

    ```typescript
    import chalk from 'chalk';
    import { bootstrap } from '../bootstrap.js';
    import type { BootstrapDeps } from '../bootstrap.js';
    import { projects } from '@get-cauldron/shared';
    import { eq, sql } from 'drizzle-orm';
    import { assetJobs } from '@get-cauldron/shared';

    interface VerifyFlags {
      json: boolean;
      projectId?: string;
    }

    export async function verifyCommand(
      args: string[],
      flags: VerifyFlags,
    ): Promise<void> {
      const subcommand = args[0];

      if (subcommand === 'assets') {
        await verifyAssets(args.slice(1), flags);
      } else {
        console.error(chalk.red('Usage: cauldron verify [assets]'));
        process.exit(1);
      }
    }
    ```

    The `verifyAssets` function runs sequential checks (following `healthCheck` pattern from `health.ts`):

    1. **Project exists and settings readable**: Query `projects` table by `flags.projectId`. Print project name and current `asset` settings. Fail if project not found.

    2. **Asset mode check**: Read `project.settings.asset?.mode`. Print current mode. Warn if `disabled` or `paused`.

    3. **ComfyUI connectivity**: HTTP GET to `COMFYUI_URL/system_stats` (default `http://localhost:8188/system_stats`) with a 3-second timeout via `AbortSignal.timeout(3000)`. Print success with GPU info if available. On failure, print actionable error: "ComfyUI not reachable at {url}. Ensure ComfyUI is running: docker compose up -d comfyui".

    4. **Database asset_jobs table accessible**: Run `SELECT COUNT(*) FROM asset_jobs WHERE project_id = $projectId`. Print job count.

    5. **Asset settings summary**: Print a summary table showing mode, runtimeUrl, artifactsRoot, maxConcurrentJobs.

    Each check prints a line: `[PASS] description` in green or `[FAIL] description` in red. If any check fails, exit with code 1 at the end.

    Parse `--real-comfyui` flag from args using `parseArgs`. When present, add step 6: submit a tiny test job (1-step, 64x64) and wait up to 60s for completion. This is opt-in and requires actual GPU.

    **Step 2 — Register in cli.ts:**

    In `packages/cli/src/cli.ts`:
    1. Add import: `import { verifyCommand } from './commands/verify.js';`
    2. Add `'verify'` to the `COMMANDS` array.
    3. Add verify BEFORE the `bootstrapClient` call (like health), since verify uses `bootstrap()` not tRPC:
       ```typescript
       if (command === 'verify') {
         await verifyCommand(commandArgs, flags);
         return;
       }
       ```
    4. Add to `printUsage()`: `console.log('  verify         Verify local asset pipeline health');`

    Note: `verify` must be handled in the same early-return block as `health`, before `bootstrapClient()` is called, because it uses `bootstrap()` directly.
